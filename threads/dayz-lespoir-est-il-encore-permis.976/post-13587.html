Il y a un gars qui a posté sur reddit (en anglais) les changements qui pourrait en découler :
Depends a little on the implementation of the game, something we who don't work with it can only speculate about. Currently their 32-bit executebles are limited to about 2GB RAM usage. What this means is that as long as you use less than 2GB RAM then there's only very negligible performance benefits from moving to 64-bit.
However if you do use more than 2GB RAM then the performance increases can be incredible. The way 32-bit applications usually deal with having to use more than 2GB RAM is by clearing memory not used right now to give space for the things you're doing right now. What this means is that the game needs to continuously re-read data from the hard-drive if it needs it again. The performance penalty of this depends a little on the implementation and on which sort of memory is thrown out. If it's things that are not needed each frame that are thrown out it, for example assets for parts of the map that is not close enough to be required to render for you, then that is kinda fine and might lead to some FPS drops if you return to that part of the map as they need to be loaded in. If more than 2GB of data is needed each frame though though then it becomes pretty problematic, and you'll get quite a heavy performance hit.
However with a proper implementation of multi-threaded asset streaming you can stream a lot of assets in and out of memory, and just simply skip rendering them if they're not loaded and get rid of all the performance problems, but instead get issues with objects popping up and some players seeing through objects that should be rendered for them etc.
Judging based on how running Arma constantly uses about 2GB of RAM for me, and how having it on an SSD will get rid of a lot of stutters I think Arma is definitely running into the 2GB limit, and I don't think Arma has any sort of multi-threaded asset streaming. However also based on how Arma actually sees a slight general FPS increase from running on an SSD I would think that it slightly does run into the 2GB limit if not every frame then every other or so. What this means is that there should be at least a small FPS increase by moving to 64-bit.
However though seeing as how RAM-restricted Arma is I'm sure they've skipped a lot of FPS optimizations that costs RAM. And maybe even done some RAM-optimizations that costs CPU cycles. So with the restriction gone they might be able to optimize a lot of things in the code to use more RAM but less CPU.
In the end it's very hard to say how it will affect the performance really, most likely it will not be for the worse though. However regardless of the performance difference it's still a really good step, easily the one that makes me the most excited out of all the changes in the roadmap, because any change to the ancient engine they have is for the better, and shows they actually have the competence to do changes with the engine (something I've been doubting a bit recently with how Arma performs and how little is done to improve it).

En gros :
Actuellement arma ne peut utiliser que 2GB de RAM maximum (et il est tout le temps au max) ce qu'il fait qui consulte régulièrement (tout le temps) votre disque pour charger les choses dont le jeu a besoin et en décharge d'autre.
Donc si ils font un exécutable 64bit ils pourront utiliser plus de RAM donc théoriquement ils pourront arrêter de faire ce qui est décrit ci dessus. (C'est pour cela que arma tourne mieux sur les ssd).
Il faut savoir aussi que si ils sont autant galère en RAM ils ont du optimiser leur jeu pour qu'il consomme le moins de RAM possible et pour ce faire utiliser plus de CPU.
=>Passage en 64bit =>Meilleur utilisation de la RAM + Moins d'utilisation CPU
Après es-ce qu'ils vont faire tout ça je n'en sais rien.

Effectivement depuis trois ans le nombre de joueurs étant intéressé par le jeu est très réduit, au mieux il y a ceux qui attendent patiemment mais la plupart comme vous visiblement sont passé à autre chose.
